Creating Instances

The following information is required to build a virtual machine instance:

Name:

A name for your virtual machine instance to help you describe and identify your machine.

Image:

An image to use for you virtual machine instance. Centos, Cirros and Ubuntu images are generally pre-loaded on your cloud.

Instance Specifications:

The size characteristics for your virtual machine. Certain images required larger sized instances.

Network:

A private network for your instance to reside on.

Security Group:

Similar to rules you would setup in a firewall to allow specific application/transport ports to connect to your instance.

Security Key:

Public and Private RSA keys that are used to access your virtual machine instance. The Public key will automatically be applied to your instance. You will use the Private key to login to your instance (if required).

Boot Options:

Booting an instance from a volume will store all instance data on physical storage. If an instance is booted ephemerally then all instance data will be lost when it is deleted.
